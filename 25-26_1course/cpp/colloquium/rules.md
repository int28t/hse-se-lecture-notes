# Коллоквиум в конце модуля A



Коллоквиум представляет собой элемент контроля индивидуального освоения знаний, умений и навыков разработки программ на языке C++ учащимися дисциплины «Основы программирования на C++» и проводится в формате устного опроса студента преподавателем с последующей выдачей задачи для реализации в виде программы на C++.



## Порядок проведения коллоквиума

Студент, приглашаемый к сдаче работы, получает последовательно *два тематических вопроса*, посвященных различным аспектам разработки программ на C++ из набора [тем](https://git.cs.hse.ru/edu/se/2025_26/cxxprog/course_materials/-/wikis/topics-refs-qs), заявленных для контроля на коллоквиуме. Преподаватель, принимающий защиту, не связан формулировками [контрольных вопросов](https://git.cs.hse.ru/edu/se/2025_26/cxxprog/course_materials/-/wikis/topics-refs-qs), представленных студентам для подготовки, может задавать уточняющие вопросы в числе и полноте, необходимых для определения уровня владения материалом, в т.ч. уточняющие вопросы по смежным темам.

После получения вопроса студент приступает к раскрытию его сути без дополнительной подготовки, в т.ч. — без подготовки письменного вспомогательного пособия (читлиста). При необходимости, ответ может поясняться студентом рисунками, диаграммами, фрагментами программного кода — приводятся непосредственно в процессе ответа. Обращение к справочной документации при раскрытии сути вопросов возможно только с разрешения преподавателя.

*Третьим элементом* коллоквиума является реализация программы на C++, которая решает задачу, выданную преподавателем. Исходные данные (dataset) предоставляются заранее и могут быть загружены студентом на персональный компьютер, на котором подразумевается выполнение задачи. Студенту рекомендуется использовать персональный компьютер (ноутбук) при наличии технической возможности. При этом необходимо заранее закрыть (выгрузить из фонового режима и т.д.) все мессенджеры, страницы веб-браузера и т.д., использование которых в рамках проекта запрещено (см. раздел «Ограничения»).

Сеанс работы с компьютером для решения задачи сопровождается видеозахватом экрана с помощью инструментов OBS Studio, Bandicam и т.п (см. раздел «Асинхронный прокторинг с OBS»). Время начала и окончания работы фиксируется. 



### Оценивание

Каждый их трех элементов коллоквиума оценивается по шкале «+-», и имеет числовой эквивалент: `+` — 3 балла, `+-` — 2 балла, `-+` — 1 балл, `-` — 0 баллов. Таким образом, максимально возможная сумма баллов 9. Полученная студентом общая сумма баллов приводится к компоненту $ C $ итоговой формулы оценки за курс по правилам, установленным программой учебной дисциплины.



### Ограничения

При сдачи коллоквиума недопустимо использовании каких-либо чатов, мессенджеров, форумов, генеративных языковых моделей и других видов интерактивных инструментов. Общение с живыми людьми в очном формате во время коллоквиума также не допускается. При выполнении задания на написание программы допустимо использование ресурса https://cppreference.com и поисковых систем, предоставляющих только ссылки на статическую информацию (документацию), например, https://duckduckgo.com/ [^ddgo]. Использование поисковых систем google, yandex и других, предоставляющих функцию подготовки ИИ-сводки, запрещено. 

[^ddgo]: Подразумевается, что поисковая система используется для поиска статических документов, с которыми разрешено ознакомляться. В случаях, когда ссылка очевидно ведет на интерактивный ресурс (например, веб-версию чата языковой модели), ее открытие не допускается.



### Асинхронный прокторинг с OBS

Загрузите, установите и правильно настройке программу захвата экрана, например, [OBS Studio](https://obsproject.com/). 

На очном коллоквиуме необходимо захватывать только экран.

Потренируйтесь заранее в работе с OBS: как начать и остановить запись, как управлять качеством/размером/нагрузкой ЦП (зависит от программного/аппаратного кодирования) и т.д. Определите место сохранения записанного видеофайла. Убедитесь, что на диске достаточно места и оно не закончится во время теста. Во время теста общение с другими людьми запрещено, предупредите заранее тех, кто будет с вами рядом на месте проведения теста.

Если вы выполнили работу до истечения отведенного на тест времени, сообщите об этом преподавателю/ученому ассистенту и следуйте его/ее указаниям. Общение с другими участниками запрещено до окончания теста. В этом случае оба участника будут дисквалифицированы.

#### После окончания написания задачи

После окончания написания задачи вам необходимо определить местоположение записанного видеофайла. Убедитесь, что он не поврежден и содержит все необходимые этапы процесса. 

Посчитайте md5-хэш-сумму полученного видео и загрузите ее в предоставленную форму. После этого изменение файла (приводящее к изменению md5) не допускается.

Далее загрузите видеофайл в любое облачное хранилище (например, Google Drive) в течение одного дня после окончания коллоквиума (чем быстрее, тем лучше, тем меньше вероятность, что вас будут критиковать), и отправьте ссылку на него в установленном порядке. Убедитесь, что файл будет доступен для скачивания не менее чем через 10 дней после дня теста. 

Для определения md5 воспользуйтесь командой для своей ОС.



[Инструкция для Windows](https://i0.wp.com/onthefencedevelopment.com/wp-content/uploads/2019/10/08acc-certutil-mysql.png?ssl=1). Пример:

```powershell
CertUtil -hashfile C:\path\to\video\file.mp4 MD5
```



[Инструкция для Linux](https://www.tecmint.com/generate-verify-check-files-md5-checksum-linux/). Пример:

```bash
md5sum file.mp4
```



[Инструкция для Mac](https://osxdaily.com/2009/10/13/check-md5-hash-on-your-mac/). Пример:

```bash
md5sum file.mp4
```
