\section{Потоки и строки STL}

\subsection{Что подразумевают под моделью ввода-вывода консольного приложения (CLI-application)? Какие основные компоненты ввода-вывода задействуются для него?}
\begin{definition}[CLI-application]
\textit{CLI-application} (\textit{Command Line Interface application}) — это программа, которая взаимодействует с пользователем через текстовый ввод и вывод в консоли.
\end{definition}

Для такой программы основными каналами являются:
\begin{itemize}
    \item \textbf{standard input} (\texttt{stdin}) — стандартный ввод;
    \item \textbf{standard output} (\texttt{stdout}) — стандартный вывод;
    \item \textbf{standard error} (\texttt{stderr}) — стандартный поток ошибок.
\end{itemize}

Обычно пользователь вводит данные с клавиатуры, а программа печатает результат в консоль. С точки зрения библиотеки STL это выражается через потоки ввода и вывода.

\subsection{Что такое стандартный поток ввода и стандартный поток вывода? Какими объектами они представлены? Какая область видимости этих объектов?}
Стандартный поток ввода в C++ представлен объектом \texttt{std::cin}, а стандартный поток вывода — объектом \texttt{std::cout}.

\begin{definition}[Стандартные потоки]
\begin{itemize}
    \item \texttt{std::cin} — объект типа \texttt{std::istream};
    \item \texttt{std::cout} — объект типа \texttt{std::ostream}.
\end{itemize}
\end{definition}

Эти объекты находятся в namespace \texttt{std} и имеют глобальную область видимости в смысле времени жизни программы: они существуют всё время выполнения программы.

\subsection{Какой заголовочный файл необходимо подключить, чтобы получить доступ к стандартным потокам ввода/вывода? Где они описаны и где определены?}
Чтобы использовать \texttt{std::cin}, \texttt{std::cout}, \texttt{std::cerr}, обычно подключают:
\begin{verbatim}
#include <iostream>
\end{verbatim}

\begin{definition}[Declaration и definition]
\textit{Declaration} сообщает компилятору о сущности, а \textit{definition} создаёт её.
\end{definition}

В случае стандартных потоков:
\begin{itemize}
    \item в заголовке \texttt{<iostream>} они \textbf{объявлены};
    \item их реальные объекты \textbf{определены} в стандартной библиотеке.
\end{itemize}

Это хороший пример различия между объявлением и определением: компилятор должен знать о существовании объекта, но сам объект создаётся библиотекой.

\subsection{Какие типы данных у стандартных потоков ввода и вывода? Какая иерархия наследования для потоковых типов STL?}
Базовые типы потоков:
\begin{itemize}
    \item \texttt{std::istream} — входной поток;
    \item \texttt{std::ostream} — выходной поток;
    \item \texttt{std::iostream} — поток, совмещающий ввод и вывод.
\end{itemize}

\begin{definition}[Иерархия потоков]
В STL потоковые классы образуют иерархию:
\begin{itemize}
    \item \texttt{std::ios\_base} — базовый уровень;
    \item \texttt{std::basic\_ios<...>} — общая часть для ввода/вывода;
    \item \texttt{std::basic\_istream<...>} и \texttt{std::basic\_ostream<...>} — отдельные ветви;
    \item \texttt{std::basic\_iostream<...>} — объединённый ввод-вывод;
    \item производные классы: \texttt{std::ifstream}, \texttt{std::ofstream}, \texttt{std::fstream}, \texttt{std::istringstream}, \texttt{std::ostringstream}, \texttt{std::stringstream}.
\end{itemize}
\end{definition}

\begin{remark}
Стандартная библиотека использует виртуальное наследование в этой иерархии, чтобы избежать проблемы «ромба» (\textit{Deadly Diamond of Death}): общая базовая часть не дублируется при наследовании по двум ветвям.
\end{remark}

\subsection{Какие операции используются для ввода и вывода? Можно ли использовать оператор ввода для потока вывода и наоборот? Как они работают с пробелами? Что возвращают?}
Для ввода используют оператор \texttt{>>}, для вывода — оператор \texttt{<<}.

\begin{itemize}
    \item \texttt{std::cin >> x;} читает данные из потока в переменную;
    \item \texttt{std::cout << x;} записывает данные в поток.
\end{itemize}

\begin{remark}
Оператор \texttt{>>} применяют к \textbf{входным} потокам, а \texttt{<<} — к \textbf{выходным}. В типичном случае их нельзя «перепутать» без ошибок типов.
\end{remark}

Для текстовых данных:
\begin{itemize}
    \item \texttt{>>} читает \textbf{токены}, разделённые пробелами;
    \item \texttt{<<} выводит строку целиком, включая пробелы.
\end{itemize}

\begin{example}
\begin{verbatim}
std::string a, b;
std::cin >> a >> b;   // reads two space-separated tokens

std::cout << "Hello, world!\n";
\end{verbatim}
\end{example}

Возвращаемое значение:
\begin{itemize}
    \item и \texttt{>>}, и \texttt{<<} возвращают \textbf{ссылку на поток};
    \item это позволяет строить цепочки: \texttt{std::cout << a << b << c;}.
\end{itemize}

\subsection{Какие функции ввода и вывода можно использовать наравне с операторами? Как прочитать целиком строку, содержащую пробелы?}
На практике вместе с операторами используют:
\begin{itemize}
    \item \texttt{std::getline()} — чтение целой строки;
    \item функции посимвольного ввода, например \texttt{get()}, \texttt{peek()}, \texttt{ignore()}.
\end{itemize}

Чтобы прочитать строку с пробелами, используют:
\begin{verbatim}
std::string line;
std::getline(std::cin, line);
\end{verbatim}

\begin{remark}
\texttt{operator>>} останавливается на пробеле, а \texttt{std::getline} читает до символа конца строки. Поэтому для текста с пробелами нужен именно \texttt{getline}.
\end{remark}

\subsection{Что такое состояние потока? Какие флаги его идентифицируют? Какие ошибки поправимы, а какие нет? Как сбросить ошибки?}
\begin{definition}[Состояние потока]
Состояние потока — это набор флагов, который показывает, можно ли продолжать чтение/запись и что пошло не так.
\end{definition}

Основные флаги:
\begin{itemize}
    \item \texttt{goodbit} — всё нормально;
    \item \texttt{eofbit} — достигнут конец файла/потока;
    \item \texttt{failbit} — операция не удалась, но поток может быть ещё пригоден;
    \item \texttt{badbit} — серьёзная ошибка потока.
\end{itemize}

По смыслу:
\begin{itemize}
    \item \textbf{рабочее состояние}: \texttt{good()};
    \item \textbf{поправимые ошибки}: обычно \texttt{failbit} и иногда \texttt{eofbit} как естественный конец чтения;
    \item \textbf{непоправимая ошибка}: \texttt{badbit}.
\end{itemize}

\begin{example}
Поправимо «сломать» поток можно попыткой прочитать число из текста:
\begin{verbatim}
int x;
std::cin >> x;   // если введено "abc", будет failbit
\end{verbatim}

Непоправимая ошибка связана с серьёзной проблемой устройства, файла или буфера.
\end{example}

Сброс флагов ошибок выполняет:
\begin{verbatim}
stream.clear();
\end{verbatim}

После ошибок форматного ввода обычно дополнительно очищают «плохой» хвост строки:
\begin{verbatim}
std::cin.clear();
std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
\end{verbatim}

\subsection{Что такое неявное приведение потока к bool? Как оно применяется в проверках вида «если поток»?}
Поток в C++ можно неявно преобразовать к булевому значению. Это удобно для проверки, можно ли продолжать работу с потоком.

\begin{definition}[Поток как bool]
Если поток преобразуется к \texttt{true}, то не установлены \texttt{failbit} и \texttt{badbit}; если к \texttt{false}, значит установлен хотя бы один из этих двух флагов.
\end{definition}

Идиоматическая проверка:
\begin{verbatim}
if (s) { ... }
if (!s) { ... }
\end{verbatim}

Важно: \texttt{if (s)} и \texttt{s.good()} — не одно и то же.
\begin{itemize}
    \item \texttt{s.good()} истинно только при состоянии \texttt{goodbit} (вообще нет флагов ошибок);
    \item булево преобразование потока эквивалентно проверке «нет \texttt{failbit} и \texttt{badbit}»;
    \item поэтому связь с \texttt{good} такая: \texttt{s.good() => bool(s)}, но не наоборот.
\end{itemize}

\subsection{Приведите идиоматические конструкции чтения до конца потока}
Для строк:
\begin{verbatim}
std::string line;
while (std::getline(std::cin, line)) {
    // process line
}
\end{verbatim}

Для токенов:
\begin{verbatim}
std::string word;
while (std::cin >> word) {
    // process word
}
\end{verbatim}

\begin{remark}
Это идиоматический стиль C++: сама операция чтения возвращает поток, и его булево значение показывает, удалось ли считать очередной элемент.
\end{remark}

\subsection{Как можно переназначить стандартные потоки ввода и вывода на другие устройства? Нужна ли перекомпиляция?}
Стандартные потоки можно перенаправить на файлы или другие устройства средствами самой операционной системы или через перенаправление в shell.

\begin{example}
\begin{verbatim}
program < input.txt > output.txt
\end{verbatim}
\end{example}

Это означает:
\begin{itemize}
    \item \texttt{stdin} берётся из файла \texttt{input.txt};
    \item \texttt{stdout} идёт в файл \texttt{output.txt}.
\end{itemize}

Перекомпиляция программы для этого не нужна.

\subsection{Что такое кодирование символов? Как решается проблема больших алфавитов? Как устроен Unicode?}
\begin{definition}[Кодирование символов]
Кодирование символов — это правило сопоставления символам числовых кодов, которые затем хранятся в памяти или передаются по каналу.
\end{definition}

Проблема национальных алфавитов состоит в том, что обычной 8-битной таблицы недостаточно для всех символов.

Unicode решает задачу так:
\begin{itemize}
    \item каждому символу соответствует кодовая позиция (\textit{code point});
    \item множество символов охватывает очень большое пространство кодов;
    \item символы группируются в плоскости (\textit{planes}).
\end{itemize}

\begin{remark}
Максимум для Unicode в современной модели — $0x10FFFF + 1 = 1\,114\,112$ кодовых позиций, то есть $2^{21}$ возможных code point-ов.
\end{remark}

\subsection{8-битные кодировки: примеры, ограничения, ASCII}
Примеры 8-битных кодировок:
\begin{itemize}
    \item ASCII как базовая часть;
    \item Windows-1251;
    \item KOI8-R;
    \item ISO-8859-1.
\end{itemize}

Ограничения:
\begin{itemize}
    \item всего $2^8 = 256$ кодовых позиций;
    \item первые 128 позиций исторически совпадают с ASCII;
    \item этого недостаточно для всех языков мира.
\end{itemize}

\begin{remark}
ASCII занимает первую половину многих 8-битных кодировок: 128 символов из 256.
\end{remark}

\begin{remark}
В стандарте Unicode символы ASCII занимают диапазон кодовых позиций
\texttt{U+0000}–\texttt{U+007F} и входят в блок \textit{Basic Latin},
который является частью \textit{Basic Multilingual Plane (BMP)}.
\end{remark}

\subsection{Что такое \texttt{std::stringstream} и когда он нужен?}
\texttt{std::stringstream} — это потоковая обёртка над строкой, которая одновременно является и входным, и выходным потоком.

\begin{itemize}
    \item \texttt{std::istringstream} — только ввод из строки;
    \item \texttt{std::ostringstream} — только вывод в строку;
    \item \texttt{std::stringstream} — и ввод, и вывод в одну и ту же строку.
\end{itemize}

\begin{example}
\begin{verbatim}
std::stringstream ss;
ss << 10 << ' ' << 20;

int a, b;
ss >> a >> b;
\end{verbatim}
\end{example}

Такие потоки удобны для парсинга, форматирования и промежуточной обработки текста без реального файла или консоли.

\subsection{Типы символов в C++ и символьные литералы}
Основные типы символов:
\begin{itemize}
    \item \texttt{char};
    \item \texttt{signed char};
    \item \texttt{unsigned char};
    \item \texttt{wchar\_t};
    \item \texttt{char16\_t};
    \item \texttt{char32\_t}.
\end{itemize}

\begin{definition}[Символьный литерал]
Символьный литерал — это запись символа в коде, например \texttt{'a'}, \texttt{'\textbackslash n'}.
\end{definition}

Префиксы уточняют тип:
\begin{itemize}
    \item \texttt{L'a'} — \texttt{wchar\_t};
    \item \texttt{u'a'} — \texttt{char16\_t};
    \item \texttt{U'a'} — \texttt{char32\_t};
    \item \texttt{u8'a'} — UTF-8-символ в кодовой форме UTF-8.
\end{itemize}

\subsection{UTF-8 как кодировка переменной длины}
UTF-8 — это кодировка переменной длины, в которой один символ Unicode кодируется от 1 до 4 байт.

Плюсы:
\begin{itemize}
    \item совместимость с ASCII;
    \item экономичное хранение латиницы;
    \item широкая поддержка в современных системах.
\end{itemize}

Минусы:
\begin{itemize}
    \item переменная длина усложняет случайный доступ;
    \item нужно аккуратно считать байты и символы;
    \item индексация по байтам не равна индексации по символам.
\end{itemize}

\subsection{Статические C-массивы, \texttt{sizeof}, decay}
\begin{definition}[C-массив]
Статический C-массив — это последовательность элементов одного типа с фиксированным количеством элементов, заданным в момент объявления.
\end{definition}

\begin{example}
\begin{verbatim}
int a[5] = {1, 2, 3, 4, 5};
\end{verbatim}
\end{example}

Число элементов можно получить как:
\[
\texttt{sizeof(a) / sizeof(a[0])}
\]

\begin{remark}
Тип массива включает число элементов. \texttt{int[5]} и \texttt{int[7]} — это разные типы.
\end{remark}

\begin{definition}[Decay]
\textit{Decay} — преобразование массива к указателю на первый элемент в большинстве контекстов.
\end{definition}

Например, массив \texttt{int a[5]} часто преобразуется к \texttt{int*}. После decay теряется информация о размере массива.

\subsection{C-строки и символ конца строки \texttt{'\textbackslash 0'}} 
\begin{definition}[C-string]
C-строка — это массив символов \texttt{char}, заканчивающийся нулевым символом \texttt{'\textbackslash 0'}.
\end{definition}

\begin{example}
\begin{verbatim}
char s[] = "hello";
\end{verbatim}
\end{example}

Здесь фактически хранится:
\[
\texttt{'h', 'e', 'l', 'l', 'o', '\textbackslash 0'}
\]

Разница между \texttt{char* arr} и \texttt{char* str} сама по себе не видна по типу: по указателю нельзя понять, ссылается ли он на обычный массив символов или на C-строку. Именно наличие \texttt{'\textbackslash 0'} делает массив строкой в стиле C.

\begin{remark}
Символ \texttt{'\textbackslash 0'} — это не символ цифры ноль \texttt{'0'}, а специальный нулевой байт со значением 0.
\end{remark}

\subsection{Класс \texttt{std::string}: назначение, инициализация, методы, внутреннее представление}
\begin{definition}[std::string]
\texttt{std::string} — класс стандартной библиотеки для хранения и обработки динамических строк.
\end{definition}

Инициализация:
\begin{verbatim}
std::string s1;
std::string s2 = "hello";
std::string s3("world");
std::string s4{"text"};
\end{verbatim}

Часто используемые методы:
\begin{itemize}
    \item \texttt{size()}, \texttt{length()};
    \item \texttt{empty()};
    \item \texttt{clear()};
    \item \texttt{append()};
    \item \texttt{substr()};
    \item \texttt{c\_str()};
    \item \texttt{push\_back()};
\end{itemize}

Метод \texttt{c\_str()} возвращает указатель на непрерывный буфер, заканчивающийся \texttt{'\textbackslash 0'}. Это нужно для взаимодействия с C-API.

\begin{remark}
Стандарт гарантирует непрерывность памяти строки и наличие завершающего нулевого символа при вызове \texttt{c\_str()}. Внутри объект может хранить данные в куче, а для коротких строк применять оптимизацию \textit{short string optimization}.
\end{remark}

\subsection{Как читать токен и строку с пробелами в \texttt{std::string}?}
Токен без пробелов:
\begin{verbatim}
std::string word;
std::cin >> word;
\end{verbatim}

Строка с пробелами:
\begin{verbatim}
std::string line;
std::getline(std::cin, line);
\end{verbatim}

\begin{remark}
После чтения через \texttt{>>} иногда перед \texttt{getline} нужно удалить символ перевода строки, например через \texttt{std::cin.ignore()}.
\end{remark}

\subsection{Связь между \texttt{sizeof(std::string)} и длиной строки \texttt{s.length()}}
\texttt{sizeof(std::string)} — это размер самого объекта-обёртки, а не длина текста внутри него.

\begin{itemize}
    \item \texttt{sizeof(std::string)} обычно постоянен для данной реализации;
    \item \texttt{s.length()} показывает число символов в строке;
    \item эти величины не обязаны быть связаны.
\end{itemize}

\begin{remark}
Короткая строка может не выходить в кучу благодаря SSO, а длинная строка почти наверняка хранится отдельно. Но \texttt{sizeof} всё равно показывает размер объекта, а не выделенной под текст памяти.
\end{remark}

\subsection{Mutable или immutable? Capacity и length}
\texttt{std::string} — это \textbf{mutable} тип: его содержимое можно менять.

\begin{itemize}
    \item \texttt{length()} — текущая длина строки;
    \item \texttt{capacity()} — объём выделенной памяти, который может быть использован без дополнительного перераспределения.
\end{itemize}

\begin{remark}
\texttt{capacity()} всегда не меньше \texttt{length()}, но может быть существенно больше.
\end{remark}

\subsection{Оптимизация коротких строк (SSO)}
\begin{definition}[Short String Optimization]
\textit{SSO} — это оптимизация, при которой короткая строка хранится прямо внутри объекта \texttt{std::string}, без отдельного выделения памяти в куче.
\end{definition}

Цель SSO:
\begin{itemize}
    \item ускорить создание и копирование коротких строк;
    \item уменьшить число обращений к динамической памяти;
    \item сократить накладные расходы.
\end{itemize}

\subsection{Преобразования строк и чисел в стиле C++}
Подходы C++:
\begin{itemize}
    \item \texttt{std::to\_string(n)} — число в строку;
    \item \texttt{std::stoi}, \texttt{std::stol}, \texttt{std::stod} — строка в число;
    \item \texttt{std::stringstream} — универсальный и гибкий механизм;
    \item \texttt{std::from\_chars}, \texttt{std::to\_chars} — низкоуровневые и эффективные функции.
\end{itemize}

\subsection{Конкатенация строк}
Менее эффективные подходы:
\begin{itemize}
    \item многократное сложение через \texttt{+} в цикле;
    \item создание большого числа временных объектов.
\end{itemize}

Более эффективные:
\begin{itemize}
    \item \texttt{append()};
    \item заранее резервировать память через \texttt{reserve()};
    \item использовать \texttt{std::stringstream} для накопления результата.
\end{itemize}

\begin{example}
\begin{verbatim}
std::string s;
s.reserve(1000);
s += "abc";
s.append("def");
\end{verbatim}
\end{example}

\subsection{\texttt{std::stringstream}: назначение, конкатенация, преобразование, тестирование}
\begin{definition}[std::stringstream]
\texttt{std::stringstream} — строковый поток, который одновременно является наследником и \texttt{std::istream}, и \texttt{std::ostream}.
\end{definition}

Основные применения:
\begin{itemize}
    \item построение строки через потоковую запись;
    \item преобразование числа в строку и обратно;
    \item тестирование функций, работающих с потоками, без файлов.
\end{itemize}

\begin{example}
\begin{verbatim}
std::stringstream ss;
ss << "x = " << 42;

std::string text = ss.str();
\end{verbatim}
\end{example}

Преобразование строки в число:
\begin{verbatim}
std::stringstream ss("123");
int x;
ss >> x;
\end{verbatim}

\begin{remark}
В модульных тестах удобно подменять файловый поток на \texttt{std::stringstream}: функция получает поток как аргумент и работает одинаково и с файлом, и со строкой в памяти.
\end{remark}

\subsection{Краткие ответы на контрольные вопросы}
\begin{itemize}
    \item \textbf{Что такое стандартный ввод и вывод?} Это \texttt{stdin} и \texttt{stdout}, в C++ обычно \texttt{std::cin} и \texttt{std::cout}.
    \item \textbf{Что такое чтение по словам?} Ввод через \texttt{>>}, который отделяет токены пробелами.
    \item \textbf{Что такое чтение строки с пробелами?} \texttt{std::getline()}.
    \item \textbf{Что такое \texttt{eof}?} Флаг конца потока, означающий достижение конца данных.
    \item \textbf{Как перебрать поток до конца?} Через \texttt{while (std::getline(...))} или \texttt{while (std::cin >> x)}.
    \item \textbf{Можно ли перенаправить ввод/вывод без перекомпиляции?} Да, через перенаправление в командной строке.
    \item \textbf{Что такое Unicode?} Стандарт кодирования большого множества символов через code point-ы.
    \item \textbf{Что такое C-строка?} Массив \texttt{char}, заканчивающийся \texttt{'\textbackslash 0'}.
    \item \textbf{Что такое \texttt{std::string}?} Удобный динамический тип для строк с управлением памятью.
    \item \textbf{Что такое SSO?} Оптимизация хранения коротких строк внутри объекта.
\end{itemize}
