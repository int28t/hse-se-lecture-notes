\subsection{Контрольные вопросы и развернутые ответы}

\subsubsection{Какие ключевые слова введены/важны (особенно с C++11 и новее) и каково их назначение?}
Ниже перечислены ключевые слова, которые особенно важны начиная с C++11, с краткой ролью:
\begin{itemize}
  \item \texttt{auto} --- автоматическое выведение типа для переменных и возвращаемых значений функций (синтаксически упрощает код, особенно с шаблонными типами).
  \item \texttt{nullptr} --- литерал нулевого указателя, безопасная замена \texttt{NULL} / \texttt{0}.
  \item \texttt{constexpr} --- константные выражения, вычисляемые на этапе компиляции при возможности; позволяет писать вычисления в контексте компиляции.
  \item \texttt{decltype} --- оператор для получения типа выражения.
  \item \texttt{override}, \texttt{final} --- спецификаторы виртуальных методов (проверка корректности переопределения, запрет дальнейшего переопределения).
  \item \texttt{thread\_local} --- спецификатор для переменных с локальным потоком хранения.
  \item \texttt{noexcept} --- спецификация, указывающая, что функция не выбрасывает исключений (может влиять на оптимизации и семантику move).
  \item \texttt{static\_assert} --- статическая проверка условий на этапе компиляции.
  \item \texttt{alignas}, \texttt{alignof} --- управление выравниванием объектов.
\end{itemize}

\subsubsection{Какова базовая структура программы на C++? Какие основные компоненты и на каком уровне видимости встречаются? Какие из них обрабатываются какими инструментами построения программы?}
Базовая структура:
\begin{itemize}
  \item \textbf{Препроцессорные директивы} (\texttt{\#include}, \texttt{\#define}) --- текстовая вставка/макросы, обрабатываются препроцессором.
  \item \textbf{Декларации и определения} (функций, классов, глобальных объектов) --- обрабатываются компилятором в единицах трансляции.
  \item \textbf{main} --- точка входа программы (функция \texttt{int main(...)}).
  \item \textbf{Пространства имён} (\texttt{namespace}) --- логическая группировка имён.
\end{itemize}

Инструменты:
\begin{itemize}
  \item Препроцессор обрабатывает \texttt{\#include} и макросы.
  \item Компилятор операции синтаксического и семантического анализа, генерации объектного кода для каждой единицы трансляции.
  \item Линкер связывает объектные модули и библиотеки.
\end{itemize}

Уровни видимости:
\begin{itemize}
  \item \textbf{scope блока} (локальные переменные).
  \item \textbf{function scope} (имена параметров, локальные метки).
  \item \textbf{namespace scope} (глобальные имена видимые в неймспейсе).
  \item \textbf{class scope} (члены класса).
\end{itemize}

\subsubsection{Какие идентификаторы корректны для объектов, типов, функций, глобальных констант, символов препроцессора? Как влияют кодстайл договорённости?}
Правила (общие):
\begin{itemize}
  \item Идентификатор начинается с буквы (A--Z, a--z, \_ (нижнее подчёркивание)); далее буквы, цифры или \_.
  \item Не может совпадать с ключевым словом.
  \item Имена, начинающиеся с одного подчёркивания и большого символа или с двух подчёркиваний, зарезервированы для реализации (стандарт).
\end{itemize}

Рекомендации по кодстайлу (проектные договорённости):
\begin{itemize}
  \item \texttt{snake\_case} для переменных/функций или \texttt{camelCase} --- выбирать одно.
  \item Типы и классы часто пишут в \texttt{PascalCase} или с префиксом (внутренние соглашения).
  \item Глобальные константы часто в \texttt{UPPER\_SNAKE\_CASE} или в \texttt{constexpr} внутри namespace.
  \item Символы препроцессора (макросы) обычно в \texttt{UPPER\_SNAKE\_CASE}.
\end{itemize}

\subsubsection{Что такое инструкция (statement)? Какой признак указывает на появление инструкции? Примеры исключений.}
\textbf{Инструкция} --- завершённое предложение языка, выполняемое во время исполнения; обычно заканчивается точкой с запятой \texttt{;}. Примеры: выражение- инструкция (\texttt{x++;}), инструкция присваивания, инструкция возврата (\texttt{return}), инструкция выбора (\texttt{if}), цикл (\texttt{for}), блок (\{\ ...\ \}).

Исключения:
\begin{itemize}
  \item Объявления (декларации) иногда не заканчиваются \texttt{;} (например \texttt{if (int x = f()) ...} --- объявление внутри заголовка \texttt{if}).
  \item Блоки сами по себе --- инструкция без завершающей точки с запятой.
\end{itemize}

\subsubsection{Что такое инструкция описания объекта? Пример.}
\textbf{Описание (declaration)} сообщает компилятору о существовании сущности без её полной реализации. Пример:
\begin{verbatim}
extern int global_value; // описание: говорит о существовании с внешней связью
\end{verbatim}

\subsubsection{Что такое инструкция описания функции? Пример.}
\textbf{Прототип (declaration)}:
\begin{verbatim}
int add(int a, int b); // прототип функции — описание интерфейса
\end{verbatim}

\subsubsection{Что такое инструкция определения и описания объекта? Пример.}
\textbf{Определение} выделяет хранилище/реализует сущность:
\begin{verbatim}
int global_value = 42; // определение и инициализация
\end{verbatim}

\subsubsection{Может ли существовать только описание без определения и наоборот? Примеры.}
Да.
\begin{itemize}
  \item Только описание: \texttt{extern int x;} --- требуется определение в другом модуле.
  \item Только определение без предварительного описания: \texttt{int x = 0;} --- это одновременно и определение, и описание.
\end{itemize}

Для функции:
\begin{verbatim}
int f(); // описание (прототип)
... (где-то else)
int f() { return 1; } // определение
\end{verbatim}

\subsubsection{Как производится инициализация объектов и когда? Варианты и различия.}
Инициализация происходит в момент определения объекта (или при создании временного объекта). Основные формы:
\begin{itemize}
  \item copy-initialization: \texttt{T x = a;}
  \item direct-initialization: \texttt{T x(a);} или \texttt{T x\{a\};}
  \item list-initialization (унифицированная): \texttt{T x\{a, b\};} или \texttt{T x = \{a,b\};}
  \item aggregate initialization: \texttt{S s\{1,2\};} для aggregate.
\end{itemize}

Общее: инициализация задаёт начальное значение/состояние объекта. Различие:
\begin{itemize}
  \item \texttt{\{\}} предотвращает narrowing (в большинстве случаев) и предпочтительна для безопасной инициализации.
  \item \texttt{= } позволяет неявные преобразования, включая narrowing при приведениях.
\end{itemize}

\subsubsection{Что такое унифицированный (list-)инициализатор? Когда рекомендуется применять?}
Унифицированный инициализатор --- это синтаксис \texttt{\{\ ...\ \}}. Рекомендуется применять для:
\begin{itemize}
  \item предотвращения нежелательного сужения (narrowing),
  \item явной инициализации aggregate-типов,
  \item единообразного стиля.
\end{itemize}

\subsubsection{Пример инструкции определения и описания с инициализацией. Когда это важно?}
\begin{verbatim}
static const int MAX = 100; // определение и инициализация (видимо глобальная константа)
struct Point { int x, y; };
Point p{1, 2}; // aggregate initialization
\end{verbatim}

Важно: для глобальных констант в заголовках использовать \texttt{inline constexpr} или \texttt{extern} + определение в .cpp, чтобы избежать множественного определения.

\subsubsection{Блок инструкций: зачем и можно ли использовать блок без заголовка?}
Блок создаёт локальную область видимости и ограничивает время жизни переменных:
\begin{verbatim}
{
    int tmp = compute();
    // tmp недоступен за пределами блока
}
\end{verbatim}
Да, блок без заголовка можно использовать для локализации временных переменных или ограничивания области видимости ресурсов (RAII).

\subsubsection{Как блок влияет на область видимости?}
Переменные, объявленные в блоке, видимы только внутри него; они уничтожаются при выходе из блока. Это позволяет контролировать время жизни и избегать конфликтов имён.

\subsubsection{Как в C++ определить именованный блок инструкций (функцию)?}
Функция --- именованный блок с параметрами:
\begin{verbatim}
int sum(int a, int b) { return a + b; }
\end{verbatim}
Параметры задают входные данные; возвращаемое значение --- выход.

\subsubsection{Объекты и типы: связь, характеристики объектов и типов.}
\begin{itemize}
  \item \textbf{Объект} --- region of storage with an associated type and value.
  \item \textbf{Тип} определяет представление и допустимые операции (семантика).
  \item Характеристики объекта: адрес (если применимо), время жизни, выравнивание, размер, значение.
  \item Характеристики типа: whether it is POD, trivial, standard-layout, signed/unsigned, arithmetic/aggregate, etc.
\end{itemize}

\subsubsection{Может ли существовать объект без типа?}
Нет в традиционном C++: каждый объект имеет тип. Исключения --- низкоуровневые конструкции (например, память, выделенная через \texttt{void*}), но сам \texttt{void*} не является объектом, это указатель на неуказанный тип.

\subsubsection{Как объекты представляются в оперативной памяти? Какие характеристики влияют?}
Представление зависит от:
\begin{itemize}
  \item типа (фундаментальный/производный/пользовательский),
  \item размера (\texttt{sizeof}),
  \item выравнивания (\texttt{alignof}),
  \item кодировки для целых/символьных типов и представления для float/double,
  \item конвенций платформы (endianness --- обычно little-endian на x86).
\end{itemize}

\subsubsection{Оператор \texttt{sizeof}: применение и возвращаемый тип}
\begin{itemize}
  \item \texttt{sizeof(T)} или \texttt{sizeof expr} возвращает значение типа \texttt{std::size\_t}.
  \item Часто используется для определения количества элементов в статическом массиве: \texttt{sizeof arr / sizeof arr[0]}.
  \item Результат обычно вычисляется на этапе компиляции (если тип известен и не VLA).
\end{itemize}

\subsubsection{Что такое операционная семантика типа? ADT примеры}
Операционная семантика --- набор операций, определённых для типа, и их поведение (например, для \texttt{int}: +, -, *, /, побитовые операции; для \texttt{std::string}: конкатенация, длина, доступ по индексу). ADT (Abstract Data Type) --- стек, очередь, словарь, множество и т.д.; примеры: \texttt{std::vector<T>} как ADT динамического массива.

\subsubsection{Классификация типов в C++ (трёхтакcонная и двухтаксонная)}
\begin{itemize}
  \item Трёхтаксонная (классическая): фундаментальные (int, char, float...), производные (указатели, массивы, ссылки, функции), пользовательские (class/struct/enum).
  \item Двухтаксонная (cppreference): фундаментальные и производные (включает пользовательские в производные/фундаментальные по контексту).
\end{itemize}

\subsubsection{Модификаторы типов для производных типов}
Примеры: указатель (\texttt{T*}), ссылка (\texttt{T\&}), массив (\texttt{T[N]}), функция типа (\texttt{T(Args...)}), cv-квалификаторы (\texttt{const}, \texttt{volatile}).

\subsubsection{Целочисленные типы: какие и как определяется знаковость, диапазоны}
\begin{itemize}
  \item Базовые типы: \texttt{char}, \texttt{short}, \texttt{int}, \texttt{long}, \texttt{long long} (со знаковостью implementation-defined для \texttt{char}).
  \item Для фиксированных размеров: \texttt{int8\_t}, \texttt{int16\_t}, \texttt{int32\_t}, \texttt{int64\_t} в \texttt{<cstdint>} (если доступны).
  \item Диапазоны можно узнать через \texttt{<climits>} (макросы) или \texttt{std::numeric\_limits<T>::min()/max()}.
  \item Знаковость: \texttt{signed} или \texttt{unsigned}; операции и преобразования зависят от этого.
\end{itemize}

\subsubsection{Типы с плавающей точкой: структура и специальные значения}
Типы: \texttt{float}, \texttt{double}, \texttt{long double}. Специальные значения: \texttt{NaN}, \texttt{+inf}, \texttt{-inf}, представляются согласно IEEE-754 на большинстве платформ.

\subsubsection{Инициализация числовых объектов, литералы, narrowing}
\begin{itemize}
  \item Литералы: \texttt{42} (int), \texttt{42u} (unsigned), \texttt{42L} (long), \texttt{3.14} (double), \texttt{3.14f} (float).
  \item Narrowing: потеря точности/диапазона при преобразовании (например, \texttt{double -> int} или \texttt{long -> int}).
  \item Списковая инициализация \texttt{\{\}} запрещает narrowing в большинстве ситуаций (даёт ошибку компиляции).
\end{itemize}

\subsubsection{Что такое POD-тип? Примеры и особенности}
POD (Plain Old Data) --- историческое понятие; в C++11 POD означает тип, который одновременно \emph{trivial} и \emph{standard-layout}. Примеры: простые структуры без пользовательских конструкторов, виртуальных функций или нестандартного размещения полей:
\begin{verbatim}
struct S { int a; double b; }; // POD в большинстве случаев
\end{verbatim}
Особенность: POD-типы совместимы с C-структурами, их можно бинарно копировать без вызова конструкторов.

\subsubsection{Что такое выражение, оператор, операнд? Может ли выражение состоять из одного операнда/оператора?}
\begin{itemize}
  \item \textbf{Выражение} --- комбинация операндов и операторов, дающая значение/эффект.
  \item \textbf{Операнд} --- объект, литерал, функция, результат другого выражения.
  \item \textbf{Оператор} --- синтаксическая конструкция, выполняющая операцию (например, \texttt{+}, \texttt{-}, \texttt{*}, \texttt{?:}).
  \item Выражение может состоять из одного операнда (например, \texttt{x}), или одного оператора в случае унарных операторов (\texttt{-x}).
\end{itemize}

\subsubsection{Классификация операторов по арности и приоритетности}
\begin{itemize}
  \item Арность: унарные, бинарные, тернарный (только \texttt{?:}).
  \item Приоритет (precedence) и ассоциативность определяют порядок вычисления; при необходимости используют скобки \texttt{(...)} для явного порядка.
\end{itemize}

\subsubsection{Охарактеризуйте оператор (на примере): возвращаемое значение и случаи отсутствия возвращаемого значения}
Разные операторы имеют разную семантику:
\begin{itemize}
  \item \texttt{sizeof} --- возвращает \texttt{std::size\_t} (имеет значение при компиляции).
  \item Присваивание (\texttt{=}) --- возвращает ссылку на левый операнд (lvalue), поэтому можно цепочить: \texttt{a = b = 0;} .
  \item Операторы управления потоком (\texttt{if}, \texttt{for}) не являются выражениями и не возвращают значение.
\end{itemize}

\subsubsection{Постройте дерево разбора для выражения (пример)}
Для выражения: \texttt{a + b * c - d} дерево разбора (с учётом приоритета \texttt{*} > \texttt{+} = \texttt{-}):
\begin{verbatim}
        (-)
       /   \
     (+)    d
    /   \
   a    (*)
        / \
       b   c
\end{verbatim}
Это показывает, что сначала вычисляется \texttt{b * c}, затем \texttt{a + (b*c)}, затем результат минус \texttt{d}.

\subsubsection{Операция присваивания. Различие между инициализацией и присваиванием}
\begin{itemize}
  \item \textbf{Инициализация} задаёт начальное значение при создании объекта: \texttt{int x = 5;} или \texttt{int x\{5\};}
  \item \textbf{Присваивание} изменяет значение уже существующего объекта: \texttt{x = 6;}
  \item Разница важна для объектов с конструкторами, для const-объектов (нельзя присваивать после инициализации) и для ресурсов (RAII).
\end{itemize}

\subsubsection{Что такое инструкция-выражение?}
Инструкция-выражение (expression-statement) --- выражение, за которым следует \texttt{;}:
\begin{verbatim}
i += 1; // инструкция-выражение
\end{verbatim}

\subsubsection{Как работает \texttt{if} (короткий, длинный, с инициализацией)?}
\begin{itemize}
  \item \textbf{Короткий:} \texttt{if (cond) stmt;}
  \item \textbf{Длинный:} \texttt{if (cond) stmt1 else stmt2;}
  \item \textbf{C++17: if с инициализацией:} \texttt{if (Type var = expr; cond(var)) stmt;}
\end{itemize}
В C++17 можно объявлять переменную прямо в header \texttt{if}, видимую только в условии и теле.

\subsubsection{Как работает \texttt{switch}? Зачем \texttt{break} и когда его не использовать?}
\texttt{switch} сравнивает целочисленное или перечислимое выражение с \texttt{case} константами. \texttt{break} предотвращает \emph{fall-through} (проваливание) к следующему case. Иногда fall-through желателен (например, группировка случаев):
\begin{verbatim}
switch (x) {
  case 1:
  case 2:
    handle_1_or_2();
    break;
  case 3:
    handle_3();
    // fall-through intentionally not used here if we want separate behavior
}
\end{verbatim}

\subsubsection{Соотношение тернарного оператора \texttt{?:} и \texttt{if}}
\begin{itemize}
  \item \texttt{?:} --- выражение, возвращает значение, удобно в контекстах, где ожидается выражение.
  \item \texttt{if} --- инструкция, позволяет выполнять сложные блоки, несколько инструкций и т.д.
  \item Ограничение \texttt{?:}: оба ветвящихся операнда должны быть совместимых типов; нельзя легко выполнять несколько инструкций без объединения в блок/функцию.
\end{itemize}

\subsubsection{Цикл \texttt{while}: назначение и особенности}
\texttt{while (cond) stmt;} --- выполняет \texttt{stmt} пока \texttt{cond} истинно. Удобно для ситуаций, где количество итераций заранее неизвестно.

\subsubsection{Цикл \texttt{for} и range-based \texttt{for}}
\begin{itemize}
  \item Классический \texttt{for}: \texttt{for (init; cond; incr) body;}
  \item Range-based (C++11): \texttt{for (auto\& elem : container) body;} --- удобен для перебора коллекций.
\end{itemize}

\textbf{Эквивалент range-based \texttt{for} классическим:}
\begin{verbatim}
std::vector<int> v = {1,2,3,4};
// range-based
for (auto &x : v) { use(x); }

// классический (итераторы)
for (auto it = v.begin(); it != v.end(); ++it) {
    auto &x = *it;
    use(x);
}

// или индексный, если допустимо:
for (std::size_t i = 0; i < v.size(); ++i) {
    auto &x = v[i];
    use(x);
}
\end{verbatim}
Требование: контейнер должен быть \emph{идентифицируемым} через \texttt{begin()/end()} (ADL допускает свободные begin/end), либо быть массивом с допустимым порядком доступа.

\subsubsection{Цикл \texttt{do..while}: особенности}
\texttt{do { body } while (cond);} --- гарантированно выполняет тело хотя бы один раз. Удобен, если условие должно проверяться после первой итерации.

\subsubsection{Может ли описание существовать без определения и наоборот?}
Уже обсуждалось: описание (declaration) может существовать отдельно (прототипы, \texttt{extern}); определение должно присутствовать при линковке (либо в текущем модуле, либо в подключаемых библиотеках).

\subsubsection{Как с помощью блока преодолеть ограничение одной инструкции, например в теле \texttt{if}?}
В теле \texttt{if} требуется одна инструкция; чтобы выполнить несколько, используют блок:
\begin{verbatim}
if (cond) {
    stmt1;
    stmt2;
}
\end{verbatim}

\subsubsection{Разница между \texttt{short} и \texttt{int} кроме размера?}
Принципиально различие --- не строгое, но важные отличия:
\begin{itemize}
  \item Семантика арифметики --- промоции целых: при арифметике типы \texttt{short}, \texttt{char} обычно продвигаются до \texttt{int} (integral promotion), что влияет на вычисления и перегрузку функций.
  \item Диапазон и поведение при переполнении (для знаковых типов переполнение неопределено).
\end{itemize}

Пример: если \texttt{short a = 30000; short b = 30000; int sum = a + b;} промежуточное поведение в \texttt{int} важно для предотвращения переполнения.

\subsubsection{Разница между \texttt{char} и \texttt{int} кроме размера?}
Да:
\begin{itemize}
  \item \texttt{char} может быть \texttt{signed} или \texttt{unsigned} в зависимости от реализации; это влияет на сравнения и расширение при промоции.
  \item \texttt{char} часто используется для хранения символов и кодировок; \texttt{int} --- для арифметики.
  \item При выражениях \texttt{char} обычно продвигается до \texttt{int} при выполнении арифметики.
\end{itemize}
Пример: \texttt{char c = '\textbackslash xFF'; if (c < 0) ...} --- поведение зависит от того, является ли \texttt{char} знаковым.

\textbf{Краткие практические советы для подготовки:}
\begin{itemize}
  \item Уметь различать описание и определение; знать роль \texttt{extern} и прототипов.
  \item На практике демонстрировать разные виды инициализации и объяснять, почему \texttt{\{\}} предпочтительнее для предотвращения narrowing.
  \item Понимать поведение \texttt{sizeof} и операции промоции целых.
  \item Быть готовым показать эквивалент range-based \texttt{for} через итераторы/индексы.
  \item Уметь объяснить, какие выражения дают значение и какие нет (операторы vs инструкции).
\end{itemize}
